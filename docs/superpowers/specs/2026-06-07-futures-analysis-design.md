# 期货基本面分析 Skill 设计文档

**创建日期**：2026-06-07
**版本**：1.0
**状态**：设计完成，待实现

---

## 1. 概述

### 1.1 目标
创建一个期货基本面分析 Skill 族，为个人投资者提供国内商品期货和能源化工品种的多维度分析能力。

### 1.2 范围
- **品种覆盖**：国内商品期货（螺纹钢、铁矿石、豆粕等）+ 能源化工（原油、PTA、甲醇等）
- **数据源**：万得（Wind）为主、公开数据源为辅
- **交互方式**：固定报告模板 + 可追问细节

### 1.3 分析能力
1. 供需平衡表分析
2. 季节性分析
3. 产量预测
4. 价格趋势预测

---

## 2. 架构设计

### 2.1 整体架构

```
用户命令 → 入口 Skill（futures-analysis）
                │
         ┌──────┼──────┬──────────┐
         ▼      ▼      ▼          ▼
      供需   季节性  产量预测  价格趋势   ← 4 个 Workflow
      工作流  工作流  工作流    工作流

        └──────┴──────┴──────────┘
                    │
                    ▼
              ┌─────────────┐
              │  数据层      │
              │  Wind MCP    │
              │  + WebFetch  │
              └─────────────┘
                    │
                    ▼
              ┌─────────────┐
              │  报告生成    │
              │  Markdown    │
              │  + 追问交互  │
              └─────────────┘
```

### 2.2 方案选型
采用**方案 C：工作流驱动**。入口 Skill 负责解析用户意图并调度对应的 Workflow，每个分析维度封装为独立 Workflow，内部通过并行 Agent 拉取多源数据后汇总。

---

## 3. 入口 Skill：`futures-analysis`

### 3.1 触发方式
`/futures-analysis` 或自然语言指令

### 3.2 职责

| 步骤 | 动作 |
|------|------|
| ① 解析 | 从用户输入提取：品种、分析类型（供需/季节/产量/趋势/全套） |
| ② 确认 | 品种或分析类型模糊时追问 |
| ③ 调度 | 启动对应 Workflow，返回报告 |

### 3.3 设计要点
- 支持简写品种名：螺纹 → 螺纹钢、TA → PTA
- 支持组合指令：可串行执行多个工作流
- 不硬编码品种列表，动态获取

---

## 4. 工作流 ①：供需平衡表

### 4.1 输入
品种名称 + 可选时间范围（默认 3 年）

### 4.2 分析维度

**供给端**：国内产量、进口量、再生/替代供给
**需求端**：下游消费量、出口量、隐性需求

### 4.3 Agent 编排

| Agent | 任务 | 数据源 |
|-------|------|--------|
| Agent 1 | 获取产量数据 | Wind MCP |
| Agent 2 | 获取进出口数据 | Wind MCP + 海关公开数据 |
| Agent 3 | 获取下游消费数据 | Wind MCP（行业产量倒推） |
| Agent 4 | 获取库存数据 | Wind MCP（社会库存、交易所仓单） |
| Synthesizer | 汇总平衡表 + 解读 | 前 4 个 Agent 结果 |

### 4.4 输出
- 供需平衡表表格（Markdown）
- 供给/需求趋势图（折线图）
- 关键结论（缺口变化、供需格局）
- 追问入口

---

## 5. 工作流 ②：季节性分析

### 5.1 输入
品种名称 + 可选年数（默认 5 年）

### 5.2 分析维度
- 月度价格季节性（历年均价对比、上涨概率、季节性因子）
- 价差季节性（近远月价差、跨品种价差、基差）
- 供需季节性（产量/消费/库存季节性指数）

### 5.3 Agent 编排

| Agent | 任务 | 数据源 |
|-------|------|--------|
| Agent 1 | 月度价格季节性 | Wind MCP K线数据 → 按月份聚合 |
| Agent 2 | 价差季节性 | Wind MCP（近远月价差、基差） |
| Agent 3 | 供需季节性 | Wind MCP 产量/消费/库存月度归一化 |
| Synthesizer | 交叉验证 + 综合结论 | 前 3 个 Agent 结果 |

### 5.4 输出
- 季节性因子图（折线图，标注极值月份）
- 当前偏离度（当前 vs 历史同期，±σ 标注）
- 交易日历（未来 3 个月季节性倾向）
- 异常预警（偏离超 2σ）

---

## 6. 工作流 ③：产量预测

### 6.1 输入
品种名称 + 预测周期（默认未来 3 个月）

### 6.2 分析维度
- 影响因素：产能利用率、检修计划、利润驱动、原料可得性、政策扰动
- 预测模型：趋势外推（基准）+ 季节性修正 + 利润驱动修正 + 综合加权

### 6.3 Agent 编排

| Agent | 任务 | 数据源 |
|-------|------|--------|
| Agent 1 | 历史产量趋势 + 产能数据 | Wind MCP |
| Agent 2 | 利润与产量关系建模 | Wind MCP（产品价格 - 原料成本） |
| Agent 3 | 季节性因子 + 检修/政策情报 | 公开数据（交易所公告、行业网站） |
| Synthesizer | 多因子加权预测 + 情景分析 | 前 3 个 Agent 结果 |

### 6.4 输出
- 未来月度产量预测表（含置信区间）
- 贡献度拆解（趋势/季节/利润贡献占比）
- 风险矩阵（上行情景/基准/下行情景）
- 历史回测误差率

---

## 7. 工作流 ④：价格趋势预测

### 7.1 输入
品种名称 + 预测周期（默认未来 1-3 个月）

### 7.2 分析维度

| 因子类别 | 指标 |
|----------|------|
| 基本面 | 供需缺口（复用①）、库存周期、利润位置（成本支撑） |
| 技术面 | 均线趋势、RSI/MACD 动量、ATR/布林带波动率 |
| 资金面 | 持仓量变化、主力合约移仓、仓单变化 |

### 7.3 Agent 编排

| Agent | 任务 | 数据源 |
|-------|------|--------|
| Agent 1 | 基本面打分 | 复用工作流①结果 + 库存数据 |
| Agent 2 | 技术面打分 | Wind MCP K线 → 多指标计算 |
| Agent 3 | 资金面打分 | Wind MCP 持仓/仓单数据 |
| Synthesizer | 加权汇总 + 综合研判 | 前 3 个 Agent 结果 |

### 7.4 输出
- 多空信号灯（看多🟢/中性🟡/看空🔴）
- 关键支撑/压力位
- 各因子得分明细表
- 情景推演（基准/乐观/悲观）
- 风险提示清单

## 8. 数据层

### 8.1 主数据源：Wind MCP
通过 `wind-mcp-skill` 获取：
- K 线行情数据
- 期货合约信息
- 产量/消费/库存基本面数据
- 持仓量/仓单数据

### 8.2 辅助数据源：公开数据
通过 `WebFetch` 获取：
- 海关进出口数据
- 交易所公告（检修计划、交割规则）
- 行业资讯网站（产能动态、政策变化）
- 国家统计局宏观数据

---

## 9. 报告格式

### 9.1 基础报告模板
- **标题**：品种 + 分析类型 + 日期
- **摘要**：3-5 条核心结论
- **详细分析**：分维度展开
- **图表**：折线图/柱状图内嵌
- **风险提示**：关键风险点
- **追问建议**：可进一步深挖的方向

### 9.2 交互设计
报告输出后，用户可：
- 追问某个具体数据来源
- 要求展开某个分析维度
- 切换时间范围重新生成
